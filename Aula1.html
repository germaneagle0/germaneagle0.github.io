Representação de números no computador

Número inteiros e reais são representado em um computador na base dois.

Um número inteiro é simples, cada posição é uma potência de dois, igual na base decimal cada posição representa uma potência de dez.

Para números reais na base dez, com parte fracionária, usa potências negativas de dez. Já na base dois são potências negativas de dois.

53dez53_{dez}53dez​ = 5×1015\times10^15×101 + 3×1003\times10^03×100
(10101)dois(10101)_{dois}(10101)dois​ = 1×22\times2^2×22 + 1×20\times2^0×20 + 1×2−2\times2^{-2}×2−2

Aritmética no Computador

  • De ponto fixo: representação de dígitos e posições de bits são fixas

Número inteiro

  • Numero inteiro: O primeiro bit representa o se é positivo (0) ou negativo (1), o restante é a magnitude. A representação em complementadores é para não repetir -0 e +0.

Ex.: +(53)dez(53)_{dez}(53)dez​ = +(110101)dois(110101)_{dois}(110101)dois​ = 000(...)0110101 (64 bits)

Número Real

  • Aritmética em ponto flutuante (pode variar a posição da vírgula), representando números reais.

  • Notação Logarítmica: mantissa a parte fracionária e a característica, o expoente de base dois. A potência multiplicada pela mantissa reproduz o número real.

Ex.: Vamos criar um computador, 64bits reservados para cada número:

  • Bit0 continua sendo o sinal.

  • Bit1-Bit7: o expoente inteiro na base dois (característica)

  • Bit8-Bit63: magnitude da parte fracionária do número (mantissa)

Digamos +(5.25)dez+(5.25)_{\text{dez}}+(5.25)dez​ = +(.10101)dois+(.10101)_{\text{dois}}+(.10101)dois​ (representa a mantissa, note 0 < m < 1) ⋅2+(11)dez\cdot 2^{+(11)_{\text{dez}}}⋅2+(11)dez​ (representa a parte característica) = +(.10101)dois⋅2+(11)dez+(.10101)_{\text{dois}}\cdot2^{+(11)_{\text{dez}}}+(.10101)dois​⋅2+(11)dez​

Ou seja: 0 0000011 10101000(...)000 (representação em flutante)

  • Existe diferentes maneiras de escrever o mesmo número em ponto flutuante.

  • Um número flutuante está representado na forma normalizada quando não existem zeros antes do digito 1 na parte fracionárias (d1\text{d}_1d1​ = 1).

  • Os números em ponto flutante são representados na forma normalizada.

Ex.: +(3.624)dez+(3.624)_{\text{dez}}+(3.624)dez​ = +(.11101)⋅2+(11)dois+(.11101)\cdot2^{+(11)_{\text{dois}}}+(.11101)⋅2+(11)dois​ = 0 0000011 1110100(...)0000

Overflow e Underflow

  • Dado N real, sua representação em aritmética de ponto flutuante de t dígitos é feita por truncamento ou arredondamento.

  • Este número não poderá ser representado neste sistema se o expoente e estiver fora do intervalo (m,M), onde m é o menor expoente e M o maior

  • Quando e< m há ocorrência de underflow e, quando e > M, há ocorrência de overflow.

Erro relativo máximo na representação de um número

  • Somente um numero finito de bits pode ser armazenado na mantissa

  • Automaticamente é introduzido um erro e não mais represetna o número desejado.

  • O maior erro relativo possível dessa represetanção do número a é:

Ea(max)≤2−t+1E_{a}(max) \le 2^{-t+1}Ea​(max)≤2−t+1 (truncamento)

Ea(max)≤0.5⋅2−t+1E_a(max) \le 0.5 \cdot 2^{-\text{t}+1}Ea​(max)≤0.5⋅2−t+1 (arredondamento)

Ex.: Digamos X = 0.937⋅1040.937 \cdot 10^40.937⋅104 e Y = 0.1272⋅1020.1272\cdot10^20.1272⋅102, obter X+Y usando t = 5 casas decimais.

0.93700 x10^4 + 0.001272x10410^4104 (não tá mais normalizado para somar, tem mais de 5 digitos, então o 2 é arrendodado para baixo, mas se fosse 0.001279 iria para 0.00128) ≈\approx≈ 0.93827x10410^4104 (2 vai embora, depois de operacionar, ai vc normaliza se precisar)
